# Plan 04-01: Implementar Perfis De ONG

<objective>
Implementar o ciclo de perfil de ONG no Volunteer Connect: consulta publica de ONGs aprovadas, visualizacao de detalhes publicos sem CPF/CNPJ, fluxo administrativo de aprovacao/reprovacao, motivo obrigatorio de reprovacao, reenvio apos reprovacao, bloqueio de edicao apos aprovacao e suspensao como alternativa a exclusao no MVP.
</objective>

<context>
## Decisoes obrigatorias

- Apenas `admin` aprova/reprova ONGs.
- ONG nasce como `pendente`.
- Reprovacao exige `motivo_reprovacao` com ate 2.000 caracteres.
- Registrar `analisado_por_usuario_id`, `analisado_em` e `reenviado_em`.
- ONG aprovada nao edita `cnpj`, `razao_social` nem `logo_storage_path` pelo sistema.
- CNPJ e CPF nunca aparecem em perfil publico ou listagem publica.
- Marketplace e perfis publicos sao visiveis para guest.
- Guest, ONG pendente e ONG reprovada podem visualizar, mas nao interagir.
- ONG aprovada nao age como voluntario.
- Exclusao fisica fica fora do MVP; usar `usuarios.conta_suspensa`.
</context>

<tasks>
## Task 1: Atualizar contrato local do banco para analise de ONG

**Tipo:** documentacao/schema

**Arquivos:**
- `docs/database/schema-proposto.md`
- `supabase/migrations/0002_campos_analise_ong.sql`

**Acao:**
- Documentar/adicionar os campos em `perfis_ongs`:
  - `motivo_reprovacao`
  - `analisado_por_usuario_id`
  - `analisado_em`
  - `reenviado_em`
- Adicionar constraint/documentacao para `motivo_reprovacao` com no maximo 2.000 caracteres.
- Documentar que `motivo_reprovacao` e obrigatorio quando `status_analise = reprovado`.
- Documentar que `analisado_por_usuario_id` referencia `usuarios.id` de um admin.
- Garantir que o SQL local nao executa nada remotamente e nao contem credenciais.

**Criterios de aceite:**
- O contrato do banco cobre rastreabilidade minima da analise.
- O SQL local e versionado e legivel em portugues.
- Nenhum CPF/CNPJ ficticio ou real novo e exposto fora de seed/documentacao controlada.

## Task 2: Criar modulo backend de ONGs

**Tipo:** backend

**Arquivos:**
- `backend/src/ongs/ongs.module.ts`
- `backend/src/ongs/ongs.controller.ts`
- `backend/src/ongs/ongs.service.ts`
- `backend/src/ongs/dto/analisar-ong.dto.ts`
- `backend/src/ongs/dto/reenviar-analise-ong.dto.ts`

**Acao:**
- Criar modulo `ongs`.
- Implementar consultas publicas:
  - listar apenas ONGs `aprovado` e com `conta_suspensa = false`;
  - detalhar apenas ONG aprovada e nao suspensa;
  - nunca retornar `cnpj`, CPF ou campos internos sensiveis.
- Implementar consulta da propria ONG autenticada para status de analise.
- Implementar reenvio de ONG reprovada:
  - permitido apenas para a propria ONG;
  - bloqueado se `conta_suspensa = true`;
  - permitido apenas quando `status_analise = reprovado`;
  - volta `status_analise` para `pendente`;
  - limpa `motivo_reprovacao`, `analisado_por_usuario_id` e `analisado_em`;
  - registra `reenviado_em`.

**Criterios de aceite:**
- APIs publicas nao vazam CNPJ/CPF.
- ONG pendente/reprovada/suspensa nao aparece publicamente.
- ONG suspensa nao consegue reenviar.
- ONG aprovada nao consegue editar dados aprovados pelo sistema.

**Verificacao:**
```powershell

npm --prefix backend test
```

## Task 3: Implementar fluxo admin de analise

**Tipo:** backend

**Arquivos:**
- `backend/src/ongs/ongs.controller.ts`
- `backend/src/ongs/ongs.service.ts`
- `backend/src/ongs/dto/analisar-ong.dto.ts`
- `backend/src/auth/guards/admin.guard.ts`

**Acao:**
- Criar endpoints protegidos por admin para:
  - listar ONGs pendentes;
  - listar ONGs reprovadas/aprovadas para acompanhamento;
  - aprovar ONG;
  - reprovar ONG com motivo obrigatorio.
- Validar `motivo_reprovacao`:
  - obrigatorio ao reprovar;
  - maximo 2.000 caracteres;
  - nao exigido ao aprovar.
- Ao aprovar:
  - `status_analise = aprovado`;
  - limpar `motivo_reprovacao`;
  - salvar `analisado_por_usuario_id`;
  - salvar `analisado_em`.
- Ao reprovar:
  - `status_analise = reprovado`;
  - salvar `motivo_reprovacao`;
  - salvar `analisado_por_usuario_id`;
  - salvar `analisado_em`.

**Criterios de aceite:**
- Apenas admin consegue aprovar/reprovar.
- Reprovacao sem motivo falha.
- Motivo acima de 2.000 caracteres falha.
- Dados de rastreabilidade sao gravados.

**Verificacao:**
```powershell
npm --prefix backend run lint:check
npm --prefix backend test
```

  - edicao/correcao de ONG;

- A regra e testada em service/guard.
