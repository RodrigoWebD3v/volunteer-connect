# Plan 02-01: Documentar Modelo Inicial Do Banco

<objective>
Criar a documentacao local e os arquivos SQL propostos para a fundacao do modelo de dados do Volunteer Connect, sem executar nada no Supabase remoto. </objective>

## Task 3: Escrever migration SQL local proposta

**Tipo:** documentacao

**Arquivos:**
- `supabase/migrations/0001_modelo_inicial.sql`

**Acao:**
- Criar SQL PostgreSQL compatível com Supabase para as tabelas definidas.
- Usar `uuid` como chave primaria com `gen_random_uuid()`.
- Usar `timestamptz` para datas.
- Criar `check constraints` para status e papeis em portugues.
- Criar foreign keys com `on delete` coerente:
  - perfis dependem de usuarios.
  - oportunidades dependem de ONGs.
  - votos/inscricoes dependem de usuario e oportunidade.
  - eventos dependem de oportunidade.
  - participacoes dependem de evento e inscricao/voluntario conforme definido no documento.
- Criar constraints de unicidade:
  - `usuarios.email`
  - `perfis_voluntarios.usuario_id`
  - `perfis_voluntarios.cpf`
  - `perfis_ongs.usuario_responsavel_id`
  - `perfis_ongs.cnpj`
  - `votos_oportunidades (oportunidade_id, usuario_id)`
  - `inscricoes (oportunidade_id, voluntario_usuario_id)`
- Criar indices basicos para buscas por oportunidade, ONG, cidade/estado e status.

**Criterios de aceite:**
- O SQL e legivel, comentado em portugues quando necessario e nao contem credenciais.
- O SQL nao inclui comandos que conectem ao Supabase remoto.
- O arquivo deixa claro que e uma proposta local.

**Verificacao:**
```powershell
Select-String -Path supabase/migrations/0001_modelo_inicial.sql -Pattern "create table","constraint","foreign key","check"
```
