# Phase 1 Plan 01: Backend Foundation

<objective>
Create the NestJS backend foundation in `backend/` with npm, strict TypeScript, generated test scaffolding, linting, formatting, and executable backend-local quality commands.
</objective>

<context>
Read before execution:

- `.planning/phases/01-project-foundation/01-CONTEXT.md`
- `.planning/phases/01-project-foundation/01-RESEARCH.md`
- `.planning/REQUIREMENTS.md`
- `AGENTS.md`

This plan covers the backend side of Phase 1 only. Do not create domain modules, Prisma schema, authentication, roles, NGO, opportunity, application, event, or participation logic in this plan.
</context>

<threat_model>
## Security Threat Model

**Assets:** generated backend source, package-lock, local developer commands.  
**Threats:** accidental secret commits, dependency script abuse, exposing production-like configuration in generated examples.  
**Mitigations:** do not add real credentials; use placeholder environment examples only if needed; keep generated dependencies locked by `package-lock.json`; do not add permissive CORS or authentication placeholders in this plan.
</threat_model>

  <task id="T1" type="auto">
    <title>Scaffold backend project</title>
    <files>
      <file>backend/**</file>
    </files>
    <action>
      From the repository root, create a NestJS backend in `backend/` using the Nest CLI with npm and strict TypeScript. Prefer a non-interactive command such as `npx @nestjs/cli new backend --package-manager npm --strict --skip-git`. If the CLI option syntax has changed, inspect `npx @nestjs/cli new --help` and use the equivalent flags. Preserve the generated npm scripts and generated unit test scaffold.
    </action>
    <verify>
      Test-Path backend/package.json
      npm --prefix backend run build
      npm --prefix backend test
    </verify>
    <acceptance_criteria>
      - `backend/package.json` exists.
      - `backend/src/main.ts` exists.
      - Backend uses npm and includes `package-lock.json`.
      - Backend build and generated tests pass.
    </acceptance_criteria>
  </task>
