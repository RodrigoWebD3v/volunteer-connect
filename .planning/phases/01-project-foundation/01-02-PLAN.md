# Phase 1 Plan 02: Frontend Foundation

<objective>
Create the SvelteKit frontend foundation in `frontend/` with npm, TypeScript, ESLint, Prettier, Vitest scaffolding, and executable frontend-local quality commands.
</objective>

<context>
Read before execution:

- `.planning/phases/01-project-foundation/01-CONTEXT.md`
- `.planning/phases/01-project-foundation/01-RESEARCH.md`
- `.planning/REQUIREMENTS.md`
- `AGENTS.md`

This plan covers frontend scaffolding only. Do not build product pages, domain forms, authentication screens, NGO screens, opportunity screens, or application flows in this plan.
</context>

<threat_model>
## Security Threat Model

**Assets:** generated frontend source, package-lock, local developer commands.  
**Threats:** committing real API URLs or secrets, weakening browser-side checks, creating product UI before auth/data contracts exist.  
**Mitigations:** use placeholder local environment values only; keep generated SvelteKit checks active; do not add secret-bearing `.env` files; avoid domain UI scope.
</threat_model>

<tasks>
  <task id="T1" type="auto">
    <title>Scaffold frontend project</title>
    <files>
      <file>frontend/**</file>
    </files>
    <action>
      From the repository root, create a SvelteKit project in `frontend/` using the current Svelte CLI with npm, TypeScript, ESLint, Prettier, and Vitest. Prefer the non-interactive command `npx sv create frontend --template minimal --types ts --add eslint prettier vitest --install npm`. If the CLI prompts or option syntax differs, inspect `npx sv create --help` and select the equivalent npm, TypeScript, ESLint, Prettier, and Vitest options.
    </action>
    <verify>
      Test-Path frontend/package.json
      npm --prefix frontend run check
      npm --prefix frontend test
    </verify>
    <acceptance_criteria>
      - `frontend/package.json` exists.
      - `frontend/src/routes/+page.svelte` or equivalent generated starter route exists.
      - Frontend uses npm and includes `package-lock.json`.
      - SvelteKit check and generated tests pass.
    </acceptance_criteria>
  </task>

  <task id="T2" type="auto">
    <title>Verify frontend quality commands</title>
    <files>
      <file>frontend/package.json</file>
      <file>frontend/eslint.config.*</file>
      <file>frontend/.prettierrc</file>
      <file>frontend/vite.config.*</file>
    </files>
    <action>
      Ensure the frontend has working `check`, `lint`, `format`, and `test` npm scripts. If the generated project lacks a `lint` script but has ESLint installed, add a script that runs ESLint against the SvelteKit source. If `format` writes changes, run it once and keep the formatted output.
    </action>
    <verify>
      npm --prefix frontend run check
      npm --prefix frontend run lint
      npm --prefix frontend run format
      npm --prefix frontend test
    </verify>
    <acceptance_criteria>
      - Frontend type/Svelte check command exits successfully.
      - Frontend lint command exits successfully.
      - Frontend format command exits successfully.
      - Frontend test command exits successfully.
    </acceptance_criteria>
  </task>
</tasks>
