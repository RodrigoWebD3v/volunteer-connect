# Plan 09-01: Fechar Integracao, Auth E Lacunas Criticas

<objective>
Fechar lacunas de integracao e seguranca antes da entrega do MVP. Esta fase completa cadastro, upload seguro, recuperacao de senha, guards reutilizaveis, integracao real das telas com APIs e testes automatizados das regras sensiveis.
</objective>

<context>
## Decisoes obrigatorias

- Admin nao aparece em cadastro publico.
- Frontend nunca recebe `SUPABASE_SERVICE_ROLE_KEY`.
- Frontend nao grava token em `localStorage`.
- CPF/CNPJ sao dados sensiveis e nao aparecem em paginas publicas.
- Toda escrita de dominio passa pelo backend.
- Scaffold visual existente deve virar fluxo integrado quando houver endpoint.
- Issues GitHub abertas sobre lacunas das fases 3 a 8 sao entradas desta fase.
</context>

<threat_model>
## Ameacas e mitigacoes obrigatorias

- **Conta meio criada:** compensar ou bloquear usuario Supabase se dominio/logo falhar.
- **Upload inseguro:** validar tipo, tamanho, path e bucket de logo no backend.
- **Bypass de papel:** role vem de `usuarios`, nunca de formulario/metadata editavel.
- **IDOR:** endpoints devem validar ownership em ONG, oportunidades, inscricoes e presencas.
- **Sessao falsa:** paginas server-side usam dados confiaveis do Supabase e backend.
- **Vazamento sensivel:** CPF/CNPJ/service role/motivo interno/stack trace nao aparecem fora de fluxos autorizados.
- **Testes com segredo real:** fixtures e env vars usam placeholders.
</threat_model>

## Task 1: Completar cadastro de voluntario e ONG

**Arquivos:**
- `backend/src/auth/auth.controller.ts`
- `backend/src/auth/auth.service.ts`
- `backend/src/auth/dto/auth.dto.ts`
- `frontend/src/routes/cadastro/+page.svelte`
- `frontend/src/routes/cadastro/+page.server.ts`

**Acao:**
- Separar/fortalecer contrato de cadastro de voluntario e ONG.
- Validar email lowercase, senha minima, CPF/CNPJ numericos e campos obrigatorios.
- Persistir `usuarios`, `perfis_voluntarios` e `perfis_ongs` de forma consistente.

**Criterios de aceite:**

- Senha nao e persistida no dominio.

## Task 2: Implementar upload seguro de logo da ONG

**Tipo:** backend/storage/seguranca

**Arquivos:**
- `backend/src/auth/auth.controller.ts`
- `backend/src/auth/auth.service.ts`
- `backend/src/supabase/supabase.service.ts`
- `frontend/src/routes/cadastro/+page.svelte`
- `frontend/src/routes/cadastro/+page.server.ts`

**Acao:**
- Receber logo de ONG no backend durante cadastro.
- Exigir logo para ONG.
- Aceitar apenas `image/png`, `image/jpeg` e `image/webp`.
- Limitar tamanho a 2 MB.
- Gerar path controlado pelo backend em `SUPABASE_LOGOS_BUCKET`.
- Salvar `logo_storage_path` em `perfis_ongs`.
- Nunca usar nome bruto do arquivo como path final.

**Criterios de aceite:**
- HARD-01 e HARD-06 cobertos para upload.
- Service role nao aparece no frontend.
- Path traversal e sobrescrita indevida ficam bloqueados.

**Verificacao:**
```powershell
npm --prefix backend test
Select-String -Path backend/src/**/*.ts -Pattern "SUPABASE_LOGOS_BUCKET","image/png","image/jpeg","image/webp"
```
