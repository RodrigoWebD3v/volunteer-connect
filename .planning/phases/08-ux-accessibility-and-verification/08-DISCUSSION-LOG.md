# Phase 8: UX, Accessibility, And Verification - Discussion Log

**Date:** 2026-05-10
**Mode:** Discussao em portugues antes de implementar

## Resumo

A fase 8 foi definida como fechamento do MVP: marketplace como primeira tela, validacao das jornadas principais, acessibilidade minima, checklist documental local e testes automatizados/E2E.

## Pergunta consolidada

Foi perguntado como a validacao final do MVP deveria funcionar, cobrindo:

- primeira tela;
- jornadas obrigatorias;
- criterios minimos de acessibilidade;
- tipo de verificacao final;
- copy institucional inicial.

## Respostas confirmadas

### Primeira tela

- A primeira tela deve ser o marketplace de oportunidades.
- Pode haver uma explicacao curta, mas o foco inicial e descobrir oportunidades.

### Jornadas obrigatorias

- Guest descobre oportunidade.
- Voluntario se cadastra e se inscreve.
- ONG se cadastra, envia perfil para analise e cria oportunidade.
- Admin aprova ONG.

### Acessibilidade

- Confirmado o padrao minimo:
  - navegacao por teclado;
  - labels em formularios;
  - contraste adequado;
  - layout responsivo mobile/desktop;
  - mensagens de erro claras.

### Verificacao

- A verificacao final deve ter os dois formatos:
  - checklist documental local;
  - testes automatizados/E2E para fluxos principais.

### Copy

- Copy simples no primeiro momento.
- Ideia base: conectar voluntarios a ONGs com oportunidades reais e organizadas.

## Decisoes capturadas

- Marketplace e a primeira experiencia publica do MVP.
- A fase deve validar UX-01..UX-04.
- Todos os requisitos v1 devem ficar verificados ou explicitamente bloqueados.
- O design segue Airbnb no publico e Notion/Cal.com em areas logadas.
- Tudo deve continuar em portugues.

## Ideias adiadas

- Landing page separada.
- Copy institucional refinada.
- Auditoria avancada de acessibilidade.
- Regressao visual completa.
