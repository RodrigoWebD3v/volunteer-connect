# Research Summary: Volunteer Connect

## Stack

Use SvelteKit, NestJS, TypeScript, and Supabase remoto com PostgreSQL gerenciado. This supports a clean academic MVP with clear module boundaries while reducing local database setup friction.

## Table Stakes

Authentication, role-based access, NGO profiles, opportunity listings, applications, schedules/events, participation history, and a simple user-friendly interface are required to validate the product.

## Watch Out For

The highest risks are unclear roles, an underspecified data model, overbuilding chat, and letting documentation drift away from implementation.

## Roadmap Implications

Start with project setup, then database modeling, then auth, then NGO profile management, then opportunity/application flows, then participation/history and polish.
